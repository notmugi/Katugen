# katugen

*matugen, but for KDE.*

Right-click any image in **Dolphin** тЖТ **Generate** тЖТ **Light** or **Dark**, and a
full Material You colorscheme is generated by [matugen][matugen] and applied
across every app you actually have installed: KDE Plasma, Alacritty / kitty /
foot / ghostty / wezterm, GTK 3/4, Qt5ct/Qt6ct, btop, helix, yazi, zathura,
cava, BetterDiscord, Vesktop, VS Code, Zed, Spicetify, Hyprland, Niri, Sway,
labwc, Mango, Scroll, Hyprtoolkit, fuzzel, walker, vicinae, Noctalia shell,
Zen Browser, Steam, Telegram Desktop, Emacs тАФ and Firefox via pywalfox (which
is auto-refreshed on every run).

Templates are migrated from [noctalia-shell][noctalia] (with one syntax fix
for matugen 4.x).

[matugen]: https://github.com/InioX/matugen
[noctalia]: https://github.com/Ly-sec/noctalia-shell

## What it does

A Dolphin service menu:

```
Right-click image  тЖТ  Generate  тЖТ  тФЬтФА Light
                                   тФФтФА Dark
```

Each click runs `matugen image --mode <light|dark> --prefer saturation`, then:

1. Sets the KDE wallpaper.
2. Writes a fresh colorscheme to every app you have installed.
3. Bounces Plasma off another scheme and re-applies **Matugen** so KDE actually
   refreshes.
4. Flips `gsettings color-scheme` for GTK apps.
5. Runs `pywalfox update` + `pywalfox <light|dark>` so Firefox refreshes live.
6. Pops a `notify-send` toast when done.

## Auto-detection (the nice part)

You don't pick which integrations to enable. The installer reads
[`config/templates.tsv`](config/templates.tsv) тАФ a registry of "if this dir
exists, install that template" rules тАФ and only wires up apps you actually
have. Re-run `./install.sh` after installing a new app and it'll be picked up
automatically.

Two templates are *always* installed regardless of detection:

- the KDE color scheme (the whole point of katugen)
- the pywalfox cache file (`~/.cache/wal/colors.json`)

## Requirements

- **KDE Plasma 6** (Plasma 5 should also work тАФ the service menu spec is the same)
- **matugen** тЙе 4.x тАФ `yay -S matugen-bin` on Arch
- `realpath` (from coreutils)
- *Optional but recommended:* `plasma-apply-colorscheme`, `plasma-apply-wallpaperimage`, `notify-send`, `gsettings`, `pywalfox`, `rsync`

## Install

```sh
git clone https://github.com/YOUR_USERNAME/katugen.git
cd katugen
./install.sh
```

The installer is idempotent тАФ re-run it any time. It only overwrites files
whose content actually changed, backing up the old version to
`*.bak-YYYYmmddHHMMSS`.

It places:

```
~/.config/matugen/templates/                       (all template files)
~/.config/matugen/config.toml                      (generated; lists only your installed apps)
~/.local/bin/matugen-generate.sh                   (the generator)
~/.local/share/kio/servicemenus/matugen-generate.desktop   (Dolphin menu)
```

## One-time per-app setup

For most apps the generated file is picked up automatically the next time the
app reads its config. A few need to be told once where to look:

- **Alacritty** тАФ in `~/.config/alacritty/alacritty.toml`:
  ```toml
  import = ["~/.config/alacritty/themes/matugen.toml"]
  ```
- **btop** тАФ `Esc тЖТ Options тЖТ Color theme тЖТ matugen`
- **Qt apps** тАФ in `qt5ct` / `qt6ct`, set *Color Scheme тЖТ matugen*
- **BetterDiscord / Vesktop** тАФ enable `matugen-material.theme.css` (Noctalia
  Material, compact) or `matugen-midnight.theme.css` (Noctalia Midnight)
- **KDE** тАФ auto-applied on each run. Manual override: *System Settings тЖТ Colors тЖТ Matugen*
- **Firefox** тАФ install the [pywalfox add-on](https://github.com/Frewacom/pywalfox-native)
  and run `pywalfox install` once. After that katugen runs `pywalfox update`
  + `pywalfox light`/`dark` on every regeneration тАФ Firefox refreshes live.
- **kitty / ghostty / wezterm / foot / starship** тАФ `include` or
  `config-file`-import the generated file from your main config.

## Uninstall

```sh
./uninstall.sh           # removes script + service menu
./uninstall.sh --purge   # also removes ~/.config/matugen/{config.toml,templates}
```

## Customizing

- **Add a new app** тЖТ drop a matugen-syntax template into `templates/` and
  add a row to `config/templates.tsv`.
- **Change an existing template** тЖТ edit the file under `templates/` and
  re-run `./install.sh` (or just edit `~/.config/matugen/templates/<file>`
  directly).
- **Change an output path** тЖТ edit `config/templates.tsv` and re-run.

Template syntax reference: [matugen template docs][mtmpl].

[mtmpl]: https://github.com/InioX/matugen/blob/main/docs/configuration/templates.md

## Troubleshooting

- **Menu doesn't appear in Dolphin** тАФ run `kbuildsycoca6 --noincremental`,
  or log out + back in. Make sure the `.desktop` file is executable.
- **`matugen` errors about "multiple source colors"** тАФ the script already
  passes `--prefer saturation`; keep that flag if you customize.
- **KDE colors don't refresh** тАФ close + reopen the affected app; KDE only
  re-reads the color scheme on app startup for most widgets.
- **Per-run log** тАФ `~/.cache/matugen-generate.log`.
