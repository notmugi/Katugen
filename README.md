# matugen-kde-dolphin

Right-click any image in **Dolphin** → **Generate** → **Light** or **Dark**, and a full
Material You colorscheme is generated by [matugen][matugen] and applied across your
desktop: KDE Plasma, Alacritty, GTK 3/4, Qt5ct/Qt6ct, btop, BetterDiscord, Vesktop,
and Firefox via pywalfox.

Templates are migrated from the excellent [noctalia-shell][noctalia] template set,
adjusted to be compatible with current `matugen` (4.x).

[matugen]: https://github.com/InioX/matugen
[noctalia]: https://github.com/Ly-sec/noctalia-shell

## What you get

A Dolphin service menu:

```
Right-click image  →  Generate  →  ├─ Light
                                   └─ Dark
```

Each click runs `matugen` against the image and writes:

| App | Output |
|---|---|
| **Alacritty** | `~/.config/alacritty/themes/matugen.toml` |
| **btop** | `~/.config/btop/themes/matugen.theme` |
| **KDE Plasma** | `~/.local/share/color-schemes/matugen.colors` (auto-applied) |
| **GTK 3** | `~/.config/gtk-3.0/matugen.css` |
| **GTK 4** | `~/.config/gtk-4.0/matugen.css` |
| **Qt 5 / Qt 6** | `~/.config/qt5ct/colors/matugen.conf`, `~/.config/qt6ct/colors/matugen.conf` |
| **BetterDiscord** | `~/.config/BetterDiscord/themes/matugen.theme.css` |
| **Vesktop** | `~/.config/vesktop/themes/matugen.theme.css` |
| **Firefox (pywalfox)** | `~/.cache/wal/colors.json` |

The wallpaper is also set via `plasma-apply-wallpaperimage`, and the GTK
light/dark preference is flipped via `gsettings`.

## Requirements

- **KDE Plasma 6** (Plasma 5 should work — the service menu spec is the same)
- **matugen** ≥ 4.x — `yay -S matugen-bin` on Arch, otherwise see the [matugen install docs](https://github.com/InioX/matugen#installation)
- `realpath` (from coreutils)
- *Optional but recommended:* `plasma-apply-colorscheme`, `plasma-apply-wallpaperimage`, `notify-send`, `gsettings`

## Install

```sh
git clone https://github.com/YOUR_USERNAME/matugen-kde-dolphin.git
cd matugen-kde-dolphin
./install.sh
```

The installer is idempotent: re-running it backs up any existing files to
`*.bak-YYYYmmddHHMMSS` before overwriting.

It places files at:

```
~/.config/matugen/templates/                       (template files)
~/.config/matugen/config.toml                      (matugen config)
~/.local/bin/matugen-generate.sh                   (the generator)
~/.local/share/kio/servicemenus/matugen-generate.desktop   (Dolphin menu)
```

…then rebuilds the KIO service cache so Dolphin picks up the menu immediately
(no logout needed).

## One-time per-app setup

After the first generation run, point each app at its `matugen` theme file once:

- **Alacritty** — in `~/.config/alacritty/alacritty.toml`:
  ```toml
  import = ["~/.config/alacritty/themes/matugen.toml"]
  ```
- **btop** — `Esc → Options → Color theme → matugen`
- **Qt apps** — in `qt5ct` / `qt6ct`, set *Color Scheme → matugen*
- **BetterDiscord / Vesktop** — enable `matugen.theme.css` in the themes panel
- **KDE** — *System Settings → Colors → Matugen* (also auto-applied each run)
- **Firefox** — install the [pywalfox add-on](https://github.com/Frewacom/pywalfox-native) and the `pywalfox` helper, then run `pywalfox install` once. After that the generator script automatically runs `pywalfox update` and `pywalfox light`/`pywalfox dark` on every regeneration, so Firefox theming refreshes live (no manual reload needed).

## Uninstall

```sh
./uninstall.sh           # removes script + service menu
./uninstall.sh --purge   # also removes ~/.config/matugen/{config.toml,templates}
```

Generated outputs (e.g. `~/.local/share/color-schemes/matugen.colors`) are
never removed — delete those manually if you want a clean slate.

## Customizing

Templates live in `templates/`. Each file uses matugen's `{{colors.NAME.default.hex}}`
syntax — see the [matugen template docs][mtmpl] for the full color list. After
editing, either run `./install.sh` again or copy the file directly into
`~/.config/matugen/templates/`.

To wire up additional apps, add a `[templates.<name>]` block to
`config/matugen-config.toml` (the `@TEMPLATES@` placeholder is replaced with
the install path).

[mtmpl]: https://github.com/InioX/matugen/blob/main/docs/configuration/templates.md

## Troubleshooting

- **The menu doesn't appear in Dolphin** — Run `kbuildsycoca6 --noincremental`,
  or log out and back in. Verify the `.desktop` file is executable.
- **`matugen` errors about "multiple source colors"** — the script already passes
  `--prefer saturation`; if you customize the script, keep that flag.
- **KDE colors don't refresh** — close and reopen the apps; KDE only re-reads
  the color scheme on app startup for most widgets. The script does bounce off
  another scheme to force Plasma's own re-read.
- **Logs** — every run appends to `~/.cache/matugen-generate.log`.
